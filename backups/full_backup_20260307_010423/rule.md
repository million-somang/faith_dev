### 4. 코드를 완성한 후 테스트 확인을 꼭 해줘.

### 5. 코드 작성을 할 때 포털 플랫폼 폴더 구조 (Monorepo)를 가지도록 해주고, apps 폴더 아래에 컨텐츠를 넣어서 Sub-App 구조로 만들어 지도록 해.

** rule.md 파일을 지켜서 작성을 해줘.

### 6. 서버를 실행(npm run dev)할 때 포트 충돌 및 라우팅 에러 방지를 위해 다음을 반드시 확인하고 조치해줘:
   - (1) 각 앱 폴더(`apps/main-portal`, `apps/app-news` 등)에 남아있는 빌드 찌꺼기인 `vite.config.js` 파일이 있다면 반드시 삭제할 것 (설정은 `vite.config.ts`만 남겨야 함).
   - (2) `vite.config.ts` 안의 `server` 객체에 `strictPort: true` 설정이 지켜지고 있는지 확인할 것. (포트가 밀리는 현상 방지)
   - (3) 기존에 5000번, 5001번, 4000번 등 백그라운드에서 돌아가고 있는 기존 node 프로세스가 없는지 반드시 확인(kill)하고 깨끗한 상태에서 서버를 시작할 것.

### 7. 미니앱 코드를 만들거나 수정하는 경우에는 MINI_APP_GUIDE.md 파일을 반드시 참고하여 작성해줘.
